# P0001 Bootstrap Flexsimulation repository

## Purpose

Create the initial repository structure for Flexsimulation as a package-driven modelling prototype.

## Scope

This package establishes:

- README repository contract,
- AGENTS workflow instructions,
- memory bootstrap files,
- requirements package model,
- documentation areas,
- configuration area,
- Python package skeleton,
- tests area,
- repository file index.

## Design intent

Flexsimulation should follow the disciplined source-of-truth style used in adjacent modelling repositories, while remaining focused on virtual-battery business-case simulation.

The first implementation should prioritise:

- reality proxy vs observation boundary,
- 15-minute baseline and activation accounting,
- rebound position ledger,
- scenario comparison,
- value attribution by capability.

## Acceptance criteria

- New AI/Codex sessions have a mandatory bootstrap path.
- The repo has a clear public-safe boundary.
- The repo uses a package-driven change model.
- The initial docs define capability, observation and business-case principles.
- A minimal Python package skeleton exists.
- `REPOSITORY_FILES.md` lists tracked files created by this package.
