# Project overview

Flexsimulation is a generic modelling prototype and future interactive executive playground for quantifying the business value of integrated distributed energy capabilities.

The core question is:

```text
How much additional value is created when distributed customer resources are planned, risk-adjusted and monetised as one integrated virtual battery instead of being optimised asset by asset?
```

The product purpose is broader than a calculation engine. It should replace static presentation and spreadsheet business cases with an interactive page where decision makers can change assumptions and discover why system capabilities and system ownership matter.

## Strategic problem

Local flexibility initiatives can produce false-positive business cases when they optimise one asset type or one partner solution in isolation. They may show gross revenue while excluding rebound, imbalance, wholesale position impact, customer constraints, cross-asset effects and future portfolio control.

The simulator should make this visible by comparing:

```text
local gross value
vs
integrated net portfolio value
```

## Intended insight

Users should be able to explore fragmented and integrated scenarios and discover that:

- edge capabilities create raw flexibility potential,
- system capabilities convert potential into profitable and risk-adjusted portfolio value,
- ownership of the system core protects value attribution, rebound handling, market integration and customer promise,
- customer value and retention improve when the product works as a coherent energy system.

The simulator is a business-case and insight machine first and an energy simulator second.

## Primary outputs

Primary outputs should include:

- incremental value vs fragmented optimisation,
- SEK/customer/year,
- customer value per customer/year,
- portfolio up/down capacity by confidence level,
- rebound cost and value,
- wholesale or imbalance impact,
- delivery success and error,
- value attribution by capability,
- strategic control or stickiness indicators.

The simulator should use synthetic customers, synthetic assets, synthetic prices and synthetic activation events unless explicit public-safe data is introduced later.
